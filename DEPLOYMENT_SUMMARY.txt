## 🚀 DEPLOYMENT OPTIONS

### Option 1: Development Mode (Quickest)
```bash

python main.py dev
# Opens: Frontend on 8501, Backend on 5000
```

### Option 2: Manual Deployment
```bash
# Terminal 1
python app_prod.py

# Terminal 2
streamlit run ui_streamlit.py
```

### Option 3: Docker
```bash
docker build -t smart-home-agent:latest .
docker run -p 5000:5000 -p 8501:8501 \
  -e MQTT_BROKER=localhost \
  smart-home-agent:latest
```

### Option 4: Docker Compose
```bash
docker-compose up -d
# Starts: Backend, Frontend, MQTT Broker
```

## 🧠 DECISION MAKING FLOW

### Autonomous Loop (Every 5 Minutes)
```
1. Context Builder collects: Temp, Humidity, Occupancy, Energy, Weather
2. Rules Engine evaluates in order:
   - Peak hour rules
   - Occupancy rules
   - Weather rules
   - Comfort rules
   - Off-peak rules
3. Generates decisions for each device
4. Executes via MQTT
5. Logs to history
```

### User Command Flow
```
1. User enters natural language or device command
2. Flask API receives request
3. SmartHomeAgent processes:
   - Try device control pattern match
   - Try information query
   - Fall back to LLM reasoning
4. Execute decision
5. Return response with decision tracking
```

## 🚨 TROUBLESHOOTING

### MQTT Connection Failed
```bash
# Check MQTT broker status
mosquitto -v

# Verify configuration
grep MQTT .env
```

### Forecast Generation Slow
```python
# Use cache (default)
forecast = forecaster.forecast_energy(use_cache=True)

# Or reduce periods
forecast = forecaster.forecast_energy(periods=6)
```

### LLM Not Available
- Check API key in `.env`
- Verify network connectivity
- Fall back to rules engine (automatic)

### API Rate Limited (HTTP 429)
- Wait 60 seconds
- Check rate limit configuration
- Reduce request frequency from client
